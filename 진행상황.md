## 프로젝트 개요

### 앱 소개
**Focus Cash** — 집중 타이머 앱. 집중할수록 크레딧을 쌓고, 크레딧으로 기프티콘/룰렛/응모를 할 수 있는 보상형 집중 앱.

### 핵심 철학
- 집중 방해 요소 제거 (일시정지 없음, 확인 팝업 없음)
- 하드코어 모드: 포기하면 크레딧 10% 차감 → 진짜 집중 유도
- 크레딧으로 실제 상품 교환 가능

## Firestore 컬렉션 구조

| 컬렉션 | 용도 | 관리 주체 |
|--------|------|----------|
| `users` | 유저 정보, 크레딧, 스트릭, 룰렛 스핀 카운트 | 앱 자동 |
| `focus_sessions` | 집중 세션 기록 | 앱 자동 |
| `credit_transactions` | 크레딧 내역 | 앱 자동 |
| `friend_requests` | 친구 요청 (status: pending/accepted/rejected) | 앱 자동 |
| `store_items` | 교환 상품 목록 | 관리자 앱 |
| `roulette_config` | 룰렛 설정 (doc: `config`) | 관리자 앱 |
| `raffle_rooms` | 응모방 | 관리자 앱 |
| `raffle_entries` | 응모 내역 (docId: `{userId}_{roomId}`) | 앱 자동 |
| `gifticon_codes` | 기프티콘 코드 | 혼합 |

### gifticon_codes prizeType 규칙
| storeItemId | 생성 경로 | prizeType |
|-------------|----------|-----------|
| `{uuid}` | 상점 직접 교환 | `'gifticon'` |
| `roulette_win` | 룰렛 크레딧 당첨 기록 | `'roulette'` |
| `raffle_{roomId}` | 응모방 기프티콘 당첨 | `'gifticon'` |
| `raffle_{roomId}` | 응모방 직접배송 당첨 | `'direct'` |

### Firestore 보안 규칙 현황 (`firestore.rules`)
- `users`: 인증 유저 읽기, 본인만 쓰기
- `focus_sessions`: 인증 유저 읽기, 본인 세션만 쓰기
- `credit_transactions`: 본인 것만 읽기/쓰기
- `friend_requests`: 발신/수신자만 읽기/쓰기
- `gifticon_codes`: 인증 유저 전체 읽기/쓰기 (count() 쿼리 때문)
- `store_items`, `raffle_rooms`, `roulette_config`: 인증 유저 전체 읽기/쓰기 (관리자 앱)
- `raffle_entries`: 인증 유저 읽기, 본인만 생성

복합 인덱스: `firestore.indexes.json` — friend_requests 4개 포함, 배포 완료
